🎯 第一步：配置 LLM 模型

⚠️ 必须先配置 LLM 模型才能使用！否则 Hermes Agent 无法响应任何请求。

部署时需通过环境变量传入 API Key（详见 GitHub README）。进入系统后，有以下三种方式配置模型：

方式一：WebUI 配置（推荐）

打开 Hermes WebUI，在设置中切换模型和提供商：

# 浏览器打开 WebUI： http://localhost:8648 # 进入 Settings → Model → 选择提供商和模型
💡 WebUI 通过 Gateway API 代理所有请求，切换模型后即时生效，无需重启。

方式二：hermes 命令行

# 交互式选择默认模型（列出所有可用模型）： $ hermes model # 完整的交互式配置向导（API Key + 模型 + 平台）： $ hermes setup # 指定模型运行单次对话： $ hermes --model anthropic/claude-sonnet-4

方式三：编辑 config.yaml

直接修改配置文件，适合高级用户或自动化场景：

$ nano /root/hermes-data/config.yaml # 核心配置项： model: default: "anthropic/claude-opus-4.6" # 默认模型 provider: "auto" # 推理提供商（auto 自动检测） # base_url: "https://openrouter.ai/api/v1" # 自定义 API 端点

💡 提示：Provider 设为 auto（默认）时，Hermes 会根据已配置的 API Key 自动选择。配置 Key 后切换模型只需 hermes model 或在 WebUI 中操作。详见 GitHub README 中的 API Key 配置说明。
⚠️ 中国用户：OpenRouter 对中国区禁用了 Google 的 API Key。如需使用 Google 模型，请在 /root/hermes-data/.env 中直接配置 GOOGLE_API_KEY。推荐使用 DeepSeek 或智谱 GLM，国内访问速度快。

❓ 常见问题

WebUI 显示"未连接"

# 1. 检查 Gateway 是否在运行并监听 8642 端口： $ curl -s http://127.0.0.1:8642/health # 2. 如果无响应，查看 Gateway 进程： $ ps aux | grep hermes # 3. 重启 Gateway： $ hermes gateway restart
WebUI "未连接"通常意味着 Gateway 未启动或 API Key 未配置。确保部署时已传入 API Key 环境变量，详见 GitHub README。

切换模型失败

在 WebUI Settings → Model 中切换模型时失败，通常是 API Key 未配置或无效。使用 hermes model 命令查看当前可用的模型列表，或检查 /root/hermes-data/.env 中的 API Key 是否正确。
