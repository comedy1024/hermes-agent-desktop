🎯 第一步：配置 LLM 模型

⚠️ 必须先配置 LLM 模型才能使用！否则 Hermes Agent 无法响应任何请求。
1
选择 LLM 提供商并获取 API Key

根据你要使用的模型，获取对应的 API Key：

🟢 OpenAI
OPENAI_API_KEY
🟣 Anthropic
ANTHROPIC_API_KEY
🔵 DeepSeek
DEEPSEEK_API_KEY
🟡 Google Gemini
GOOGLE_API_KEY
🟠 OpenRouter
OPENROUTER_API_KEY
🔴 智谱 GLM
GLM_API_KEY
🟤 Kimi
KIMI_API_KEY
⚪ 本地 Ollama
无需 Key
2
配置 API Key

在容器终端中编辑配置文件：

$ nano /root/hermes-data/.env # 添加你的 API Key（按需选择一个或多个）： OPENAI_API_KEY=sk-xxx ANTHROPIC_API_KEY=sk-ant-xxx DEEPSEEK_API_KEY=sk-xxx GOOGLE_API_KEY=AIzaXXX # ⚠️ 必须配置以下两项，否则 Gateway 拒绝所有请求： API_SERVER_ENABLED=true GATEWAY_ALLOW_ALL_USERS=true
3
重启 Gateway 并选择模型

保存 .env 后，重启 Gateway 使配置生效：

$ hermes gateway restart # 选择默认模型： $ hermes model # 或通过 WebUI 配置（推荐）： # 打开 http://localhost:8648 → Settings → Model
💡 中国用户注意：OpenRouter 对中国区禁用了 Google 的 API Key。如需使用 Google 模型，请直接配置 GOOGLE_API_KEY。推荐使用 DeepSeek 或智谱 GLM，国内访问速度快。

❓ 常见问题

WebUI 显示"未连接"

# 检查 Gateway 是否在 8642 端口监听 $ curl -s http://127.0.0.1:8642/health # 如果无响应，检查 .env 中是否有以下配置： $ grep API_SERVER_ENABLED /root/hermes-data/.env $ grep GATEWAY_ALLOW_ALL_USERS /root/hermes-data/.env # 如果缺失，添加并重启： $ echo 'API_SERVER_ENABLED=true' >> /root/hermes-data/.env $ echo 'GATEWAY_ALLOW_ALL_USERS=true' >> /root/hermes-data/.env $ hermes gateway restart

切换模型失败

确认目标模型的 API Key 已在 /root/hermes-data/.env 中正确配置，且 GATEWAY_ALLOW_ALL_USERS=true 已设置。配置后执行 hermes gateway restart。
